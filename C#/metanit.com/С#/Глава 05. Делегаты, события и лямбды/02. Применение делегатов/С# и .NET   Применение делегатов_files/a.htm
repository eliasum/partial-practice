                                account.RegisterHandler(new Account.AccountStateHandler(Show_Message)); текущий вариант.

                                Разберёмся. account -- это объект класса Account. От него вызывается RegisterHandler публичный нестатический метод, который призван определить поле делегата _del объекта account (т.е. определить account._del)
                                Тут всё ок. Но то, что идёт дальше в скобках меня пугает.

                                RegisterHandler должен определить делегат через новый делегат от AccountStateHandler, который в свою очередь должен съесть в себя метод Show_Message(). Т.е. это будет объединение делегатов account._del и нового.

                                                                                          Наведу еще один пример от себя. За базу возьмем гипотетическую пошаговую игру:

                                                                                          public class BaseUnit
                                                                                          {
                                                                                               public int hp = 100;

                                                                                               public delegate void UnitStateChanger(BaseUnit target);

                                                                                               public UnitStateChanger unitStateChanger;

                                                                                               public void EndOfMove()
                                                                                               {
                                                                                                    unitStateChanger?.Invoke(this);
                                                                                               }
                                                                                          }


                                                                                          У нас есть класс базового бойца, в которого есть возможность принять на вход некоторые методы. И если делегат не пустой, мы все его методы выполняем в конце хода.

                                                                                          public class Mage : BaseUnit
                                                                                          {
                                                                                               public static void Decay(BaseUnit target)
                                                                                               {
                                                                                                    target.hp -= 10;
                                                                                               }

                                                                                               public void ApplyDecayOnTarget(BaseUnit target)
                                                                                               {
                                                                                                    target.unitStateChanger += Decay;
                                                                                               }
                                                                                          }


                                                                                          Дальше у нас есть наследник базового бойца - маг. У мага есть метод "гниение" и метод "наложения гниения" на цель. Когда мы используем ApplyDecayOnTarget(BaseUnit target) мы передаем в делегат соперника метод "гниение", который будет запущен соперником в конце хода.
                                                                                          Пример вызова:

                                                                                          BaseUnit fighter= new BaseUnit();
                                                                                          Mage mage = new Mage();

                                                                                          mage.ApplyDecayOnTarget(fighter);

                                                                                          fighter.EndOfMove();


                                                                                          В результате в конце хода наш боец станет беднее на 10 хп. И так в конце каждого хода, пока "гниение" не будет с него снято при помощи 

                                                                                          target.unitStateChanger -= Decay;

                                                                                          Прелесть использования здесь делегата в том, что таких состояний в бойца можно набросать целую кучу и все они будут выполняться в конце хода. Это могут быть как и всякие "яды", "лечения", "усиления" и вообще всё чего душа пожелает.
