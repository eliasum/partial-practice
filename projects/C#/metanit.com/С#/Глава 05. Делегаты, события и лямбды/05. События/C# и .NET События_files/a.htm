            this - это текущий объект, то есть объект, который вызвал событие. Он передается в событие, потому что мы определили такую сигнатуру события через делегат
            public delegate void AccountStateHandler(object sender, AccountEventArgs e);

            Передача sendera позволит при обработке события получить информацию о инициаторе события. В некоторых случаях это особенно бывает важно, например, в графических программах.
            Ну и второй параметр - это сведения о событии в виде объект AccountEventArgs
            поскольку в программе за обработку события Added отвечает метод Show_Message, то фактически все эти данные будут передаваться в этот метод Show_Message

                                        Еще один пример показывающий разницу между событиями и делегатами. Попробуйте написать аналог с помощью делегатов - чтобы объект мог подписать/отписать только на свои методы - получится, но немного сложнее. Фишка именно в том, что события не позволяют прямое присваивание, а работают только через += -+
                                        (За пример прошу не ругать, писал сам для себя для прояснения моментов работы с событиями)

                                        using System;
                                        namespace ConsoleAppTests
                                        {
                                        public delegate void EvtTemplate();
                                        class Program
                                        {
                                        static void Main(string[] args)
                                        {
                                        EventProducer eventProducer = new EventProducer(); //объект, который будет генерировать событие
                                        EventSubcriber1 evtS1 = new EventSubcriber1();
                                        evtS1.registerSubscription(eventProducer); //подписываемся на событие, генерируемое в eventProducer
                                        EventSubcriber2 evtS2 = new EventSubcriber2();
                                        evtS2.registerSubscription(eventProducer);
                                        //eventProducer.evtInstance = null; // не скомпилируется - напрямую работать с событием нельзя, только через += -+
                                        //в результате evtS1 evtS2 могут подписаться на событие, отписаться от него
                                        //но не повлиять на подписку других объектов - обратите внимание на модификатор доступа "private" метода evtCatcher
                                        eventProducer.produce(); // по событию срабатывают приватные методы в подписанных evtS1, evtS2
                                        Console.WriteLine();
                                        evtS1.unregisterSubscription(eventProducer); // отписываем evtS1
                                        eventProducer.produce(); // по событию срабатывают приватные методы в подписанном evtS2
                                        Console.ReadKey();
                                        }
                                        }
                                        class EventProducer
                                        {
                                        public event EvtTemplate evtInstance; // определяем событие, в котором будем обрабатывать ссылки на методы "подписаных" объектов
                                        public void produce()
                                        {
                                        /*выполняем полезную работу*/
                                        evtInstance(); // запуск срабатывания события - уведомляем подписанные объекты о срабатывании
                                        }
                                        }
                                        class EventSubcriber1
                                        {
                                        public void registerSubscription(EventProducer eventProducerI)
                                        {
                                        eventProducerI.evtInstance += evtCatcher;
                                        }
                                        public void unregisterSubscription(EventProducer eventProducerI)
                                        {
                                        eventProducerI.evtInstance -= evtCatcher;
                                        }
                                        private void evtCatcher()
                                        {
                                        Console.WriteLine("Event catched in EventSubcriber1");
                                        }
                                        }
                                        class EventSubcriber2
                                        {
                                        public void registerSubscription(EventProducer eventProducerI)
                                        {
                                        eventProducerI.evtInstance += evtCatcher;
                                        }
                                        private void evtCatcher()
                                        {
                                        Console.WriteLine("Event catched in EventSubcriber2");
                                        }
                                        }
                                        }

                                                              Делегаты и события отличаются приблизительно, как поля и свойства
                                                              Делегаты содержат просто ссылки на методы
                                                              События работают в некотором роде аналогично свойствам, они содержат неявные методы add, remove (для реализации операций += -=) которые управляют содержимым ссылок на методы. По умолчанию они работают как у делегатов. Но в отличии от делегатов, со событиями нельзя работать прямым присваиванием, только через операции += -=.

                                                              //пример события, у которого методы явно определены
                                                              // взято из Шилдт - C# 4.0. Полное руководство. 2011
                                                              delegate void MyEventHandler();
                                                              class MyEvent
                                                              {
                                                              MyEventHandler[] evnt = new MyEventHandler[3]; //определяем контейнер для сохранения ссылок на методы
                                                              //аналог поля для свойства
                                                              //событие сможет сохранять и обрабатывать только 3 ссылки на методы
                                                              public event MyEventHandler SomeEvent
                                                              {
                                                              add // Добавить событие в список. Аналог сеттера для свойства
                                                              // Определяем операцию += для события
                                                              {
                                                              int i;
                                                              //сохраняем ссылку на метод, если сохранено меньше 3х ссылок на методы
                                                              for (i = 0; i < 3; i++)
                                                              if (evnt[i] == null)
                                                              {
                                                              evnt[i] = value;
                                                              break;
                                                              }
                                                              //пишем сообщение об заполнености списка событий
                                                              //в результате 4я операция += не добавит ссылку на метод в событие
                                                              if (i == 3) Console.WriteLine("Список событий заполнен.");
                                                              }
                                                              remove // Удалить событие из списка.
                                                              // Определяем операцию -= для события
                                                              {
                                                              int i;
                                                              //удаляем ссылку на метод из списка событий
                                                              for (i = 0; i < 3; i++)
                                                              if (evnt[i] == value)
                                                              {
                                                              evnt[i] = null;
                                                              break;
                                                              }
                                                              //пишем сообщение, если метода нет в списке событий
                                                              if (i == 3) Console.WriteLine("Обработчик событий не найден.");
                                                              }
                                                              }
                                                              }

                                                                                                                                              это нет. Смотрите, допустим, вы хотите, чтобы при вызове метода mess() выполнялось некоторое действие -там где у вас идет строка Console.WriteLine("Message"); - это может быть вывод на консоль, это может отображение графического окна с сообщением, это может быть отправка на электронную почту, это может быть все что угодно, но вы не знаете на момент определения класса, что это может быть за действие. Тем более ваш класс может использоваться другими людьми, которые сами захотят определить нужное им действие. Для этого вместо строки

                                                                                                                                              Console.WriteLine("Message");
                                                                                                                                              вызывается событие. А дейтвие, которое будет выполняться при вызове этого события, определяется непосредственно при использовании класса.

                                                                                                                                              Например, в случае с примером в статье на консоль выводится состояние условного счета. Но мы можем при использовании класса Account назначить кучу обработчиков данного события, которые будут выполнть различную работу. При этом нам не надо менять исходный код класса. Более того исходный код нам может быть в принципе недоступен, например, мы используем скомпилированную библиотеку dll, однако с помощью событий мы все равно можем добавить к событию обработчики и обработать его при его возникновении
