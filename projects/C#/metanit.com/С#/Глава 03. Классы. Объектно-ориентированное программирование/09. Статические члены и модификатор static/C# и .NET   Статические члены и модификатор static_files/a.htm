                                                                                                  public static Person SetName(string Name)

                                                                                                                                статические методы употребляются, когда нет смысла создавать объект класса для доступа к этим методам. Например, есть класс Math, и у него есть метод Sqrt(), который вычисляет квадратный корень числа. Действие этого метода зависит только от параметра и никак не зависит от внутреннего состояния класса Math. Поэтому такой метод можно сделать статическим. То есть мы можем создать класс с методами. Но если выполнение этих методов никак не зависит от того, что представляет объект класса, от значений его полей и свойств, то такие методы можно сделать статическими.

                                                                                                                                Или например, есть паттерн Синглтон, - http://metanit.com/sharp/pa... - без статических методов и полей его было бы сложно создать
                                                                                                                                Еще в .NET есть такая функциональность, как методы расширения, которые добавляют функциональность к уже существующим классам без их переопределения. И для этого также используются статические методы.

                                                                                                                                Статические классы используются, когда все методы и поля этого класса являются статическими. То есть нет смысла создавать объект класса, чтобы использовать его функциональность в виде методов
