        int x = 1;
        int y = 2;

            Хотелось бы добавить про инициализацию именованных полей кортежа при использовании явных типов:

            Если используя динамическое определение кортежа мы делаем так:


            var tuple = (x: 42, PI: Math.PI);

            Console.WriteLine(tuple.x); //42

            Console.WriteLine(tuple.PI); //3,14159265358979

            То при использовании статического это будет выглядеть так:


            (int x, double PI) tuple = (42, Math.PI);

            Console.WriteLine(tuple.x); // 42

            Console.WriteLine(tuple.PI); //3,14159265358979

            При всём этом, переменные x и PI во втором примере не будут видны снаружи. Они будут видны только в случае если убрать имя кортежа (tuple2), тогда уже получится обычное определение переменных (в жабаскрипте это называется destructuring assignment).

                                Важные отличия:
                                1) Можно легко и непринужденно объявить переменную типа ValueTuple`2 с конкретными параметрами типа.
                                Переменную анонимного типа (без присвоения ссылки на известный экземпляр) объявить нельзя.
                                2) Как уже было сказано, анонимный тип - ссылочный, ValueTuple - значимый. Это структура. Но нельзя забывать, что System.Tuple<tn> - тоже ссылочный тип.
                                3) У ValueTuple есть шикарная композиция/декомпозиция из отдельных переменных.
                                ----
                                Декомпозицию любого класса неправильно назвали Deconstruct:
                                class Point
                                {
                                public int X { get; }
                                public int Y { get; }

                                public Point(int x, int y) { X = x; Y = y; }
                                public void Deconstruct(out int x, out int y) { x = X; y = Y; }
                                }

                                (var myX, var myY) = GetPoint(); // вызывает Deconstruct(out myX, out myY);

                                Это декомпозиция.
                                -----
                                Добавлены подстановочные символы в C# 7:
                                public static Tuple\int, int\ Func() // Здесь int, int в угловых скобках. Глюк комментариев.
                                {
                                return Tuple.Create(7, 5);
                                }

                                (x, _) = Func(); // Если мы хотим не все поля кортежа использовать.

                                var list = new List<(string name, int age)>();
