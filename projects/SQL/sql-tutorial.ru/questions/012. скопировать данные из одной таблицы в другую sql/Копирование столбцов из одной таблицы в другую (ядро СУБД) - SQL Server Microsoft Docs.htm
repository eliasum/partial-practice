Перед началомBefore You Begin

ОграниченияLimitations and Restrictions

При копировании из одной базы данных в другую столбца, имеющего псевдоним типа данных, исходный тип данных в целевой базе данных может оказаться недоступным.When you copy a column that has an alias data type from one database to another, the alias data type may not be available in the destination database. В этом случае столбцу будет назначен ближайший подходящий базовый тип данных, доступный в целевой базе данных.In such a case, the column will be assigned the nearest matching base data type available in that database.

безопасностьSecurity

PermissionsPermissions

Требуется разрешение ALTER на таблицу.Requires ALTER permission on the table.

Использование среды SQL Server Management StudioUsing SQL Server Management Studio

Копирование определения столбца из одной таблицы в другуюTo copy column definitions from one table to another

  1. Щелкнув правой кнопкой мыши, откройте таблицу со столбцами, предназначенными к копированию, и таблицу, в которую необходимо выполнить копирование, затем нажмите кнопку Разработка.Open the table with columns you want to copy and the one you want to copy into by right-clicking the tables, and then clicking Design.

  2. Выберите вкладку исходной таблицы и выделите нужные столбцы.Click the tab for the table with the columns you want to copy and select those columns.

  3. В меню Правка выберите Копировать.From the Edit menu, click Copy.

  4. Выберите вкладку таблицы, в которую требуется скопировать столбцы.Click the tab for the table into which you want to copy the columns.

  5. Выделите столбец, после которого нужно поместить вставляемые столбцы, и в меню Правка выберите Вставить.Select the column you want to follow the inserted columns and, from the Edit menu, click Paste.

Копирование данных из одной таблицы в другуюTo copy data from one table to another

  1. Следуйте приведенным выше инструкциям для копирования определения столбцов.Follow the directions for copying column definitions above.

    Примечание

    Прежде чем скопировать данные из одной таблицы в другую, убедитесь, что типы данных целевых столбцов совместимы с типами данных исходных.Before you begin to copy data from one table to another, make sure that the data types in the destination columns are compatible with the data types of the source columns

  2. Откройте новое окно редактора запросов.Open a new Query Editor window.

  3. Щелкните редактор запросов правой кнопкой мыши и выберите Создать запрос в редакторе.Right-click the Query Editor, and then click Design Query in Editor.

  4. В диалоговом окне Добавление таблицы выберите исходную и целевую таблицы, нажмите кнопку Добавить, а затем закройте диалоговое окно Добавление таблицы .In the Add Table dialog box, select the source and destination table, click Add, and then close the Add Table dialog box.

  5. Щелкните правой кнопкой мыши пустую область редактора запросов, наведите указатель на пункт Изменить тип и выберите Вставить результаты.Right-click an open area of the Query Editor, point to Change Type, and then click Insert Results.

  6. В диалоговом окне Выбор целевой таблицы для инструкции Insert Results выберите целевую таблицу.In the Choose Target Table for Insert Results dialog box, select the destination table.

  7. В верхней части конструктора запросов щелкните исходный столбец исходной таблицы.In the upper portion of the Query Designer, click the source column in the source table.

  8. Конструктор запросов создал запрос INSERT.The Query Designer has now created an INSERT query. Нажмите кнопку "ОК", чтобы поместить запрос в исходное окно редактора запросов.Click OK to place the query into the original Query Editor window.

  9. Выполните запрос, чтобы вставить данные из исходной таблицы в целевую.Execute the query to insert the data from the source table to the destination table.

Использование Transact-SQLUsing Transact-SQL

Копирование определения столбца из одной таблицы в другуюTo copy column definitions from one table to another

  1. Нельзя копировать отдельные столбцы из одной таблицы в другую (существующую) с использованием инструкций Transact-SQL.You cannot copy individual columns from one table to another existing table by using Transact-SQL statements. Однако можно создать новую таблицу в файловой группе по умолчанию и вставить в нее результирующие строки из запроса с помощью инструкции SELECT INTO.However, you can create a new table in the default filegroup and inserts the resulting rows from the query into it by using SELECT INTO. Дополнительные сведения см. в разделе Предложение INTO (Transact-SQL).For more information, see INTO Clause (Transact-SQL).

Копирование данных из одной таблицы в другуюTo copy data from one table to another

  1. В обозревателе объектов подключитесь к экземпляру компонента Компонент Database EngineDatabase Engine.In Object Explorer, connect to an instance of Компонент Database EngineDatabase Engine.

  2. На стандартной панели выберите пункт Создать запрос.On the Standard bar, click New Query.

  3. Скопируйте следующий пример в окно запроса и нажмите кнопку Выполнить.Copy and paste the following example into the query window and click Execute.

    USE AdventureWorks2012;  
GO  
CREATE TABLE dbo.EmployeeSales  
( BusinessEntityID   varchar(11) NOT NULL,  
  SalesYTD money NOT NULL  
);  
GO  
INSERT INTO dbo.EmployeeSales  
    SELECT BusinessEntityID, SalesYTD   
    FROM Sales.SalesPerson;  
GO
