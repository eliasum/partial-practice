Приоритет операторов (Transact-SQL)Operator Precedence (Transact-SQL)

  • 15.03.2017
  • Чтение занимает 2 мин

Если сложное выражение содержит множество операторов, порядок выполнения операций определяется приоритетом операторов.When a complex expression has multiple operators, operator precedence determines the sequence of operations. Порядок исполнения может существенно повлиять на результирующее значение.The order of execution can significantly affect the resulting value.

Уровни приоритета операторов показаны в следующей таблице.Operators have the precedence levels shown in the following table. Оператор с более высоким уровнем выполняется прежде, чем оператор с более низким уровнем.An operator on higher levels is evaluated before an operator on a lower level. В следующей таблице самым высоким уровнем является 1, а самым низким — 8.In the following table, 1 is the highest level and 8 is the lowest level.

Таблица 1
LevelLevel ОператорыOperators
11 ~ (побитовое НЕ)~ (Bitwise NOT)
22 * (умножение), / (деление), % (остаток деления)* (Multiplication), / (Division), % (Modulus)
33 + (положительное), – (отрицательное), + (сложение), +( объединение), – (вычитание), & (побитовое И), ^ (побитовое исключающее ИЛИ), | (побитовое ИЛИ)+ (Positive), - (Negative), + (Addition), + (Concatenation), - (Subtraction), & (Bitwise AND), ^ (Bitwise Exclusive OR), | (Bitwise OR)
44 =, >, <, >=, <=, <>, !=, !>, !< (операторы сравнения)=, >, <, >=, <=, <>, !=, !>, !< (Comparison operators)
55 NOTNOT
66 ANDAND
77 ALL, ANY, BETWEEN, IN, LIKE, OR, SOMEALL, ANY, BETWEEN, IN, LIKE, OR, SOME
88 = (присваивание)= (Assignment)

Если два оператора в выражении имеют один и тот же уровень приоритета, они вычисляются в порядке слева направо по мере их появления в выражении.When two operators in an expression have the same precedence level, they're evaluated left to right based on their position in the expression. Например, если выражение использует инструкцию SET, то оператор вычитания будет выполнен до оператора сложения.For example, in the expression that is used in the following SET statement, the subtraction operator is evaluated before the addition operator.

SQL
DECLARE @MyNumber INT;  
SET @MyNumber = 4 - 2 + 27;  
-- Evaluates to 2 + 27 which yields an expression result of 29.  
SELECT @MyNumber;

Чтобы изменить приоритет операторов в выражении, следует использовать скобки.Use parentheses to override the defined precedence of the operators in an expression. Все находящееся внутри скобок вычисляется для получения одного значения.Everything within parentheses is evaluated to yield a single value. Это значение может быть использовано любым оператором за пределами скобок.That value can be used by any operator outside those parentheses.

Например, если выражение использует инструкцию SET, то у оператора умножения более высокий приоритет, чем у оператора сложения.For example, in the expression used in the following SET statement, the multiplication operator has a higher precedence than the addition operator. Оператор умножения вычисляется первым, и результатом выражения будет 13.The multiplication operation is evaluated first; the expression result is 13.

SQL
DECLARE @MyNumber INT;  
SET @MyNumber = 2 * 4 + 5;  
-- Evaluates to 8 + 5 which yields an expression result of 13.  
SELECT @MyNumber;

В выражении следующей инструкции SET скобки показывают, что сначала нужно вычислить операцию сложения.In the expression used in the following SET statement, the parentheses cause the addition to be evaluated first. Результатом выражения будет 18.The expression result is 18.

SQL
DECLARE @MyNumber INT;  
SET @MyNumber = 2 * (4 + 5);  
-- Evaluates to 2 * 9 which yields an expression result of 18.  
SELECT @MyNumber;

Если в выражении содержатся вложенные скобки, то сначала вычисляется результат наиболее глубоко вложенных скобок.If an expression has nested parentheses, the most deeply nested expression is evaluated first. В следующем примере показано использование вложенных скобок, в наиболее глубоко вложенных скобках записано выражение 5 - 3.The following example contains nested parentheses, with the expression 5 - 3 in the most deeply nested set of parentheses. Результатом этого выражения будет 2.This expression yields a value of 2. После этого оператор сложения (+) добавляет этот результат к 4, чтобы получить значение 6.Then, the addition operator (+) adds this result to 4 which yields a value of 6. Наконец, 6 умножается на 2, чтобы получить результат выражения 12.Finally, the 6 is multiplied by 2 to yield an expression result of 12.

SQL
DECLARE @MyNumber INT;  
SET @MyNumber = 2 * (4 + (5 - 3) );  
-- Evaluates to 2 * (4 + 2) which then evaluates to 2 * 6, and   
-- yields an expression result of 12.  
SELECT @MyNumber;
