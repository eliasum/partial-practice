Учебник по Node.js в коде Visual Studio

Node.js - это платформа для создания быстрых и масштабируемых серверных приложений с использованием JavaScript. Node.js - это среда выполнения, а npm - менеджер пакетов для модулей Node.js.

Visual Studio Code имеет встроенную поддержку языков JavaScript и TypeScript, а также отладку Node.js. Однако для запуска приложения Node.js вам потребуется установить среду выполнения Node.js.

Чтобы начать в этом пошаговом руководстве, установите Node.js для вашей платформы . Диспетчер пакетов Node включен в дистрибутив Node.js. Вам нужно будет открыть новый терминал (командную строку), чтобы инструменты nodeи npmкомандной строки находились в вашей переменной PATH.

Linux : Существуют специальные пакеты Node.js для различных разновидностей Linux. См. Установка Node.js через менеджер пакетов, чтобы найти пакет Node.js и инструкции по установке, адаптированные к вашей версии Linux.

Совет: Чтобы проверить, правильно ли установлен Node.js на вашем компьютере, откройте новый терминал и введите текст, node --helpи вы должны увидеть документацию по использованию.

Привет, мир#

Давайте начнем с создания простейшего приложения Node.js «Hello World».

Создайте пустую папку с именем "hello", перейдите и откройте VS Code:

mkdir hello
cd hello
code .

Совет: вы можете открывать файлы или папки прямо из командной строки. Период '.' относится к текущей папке, поэтому VS Code запустится и откроет Helloпапку.

На панели инструментов проводника нажмите кнопку «Новый файл»:

и назовите файл app.js:

Используя .jsрасширение файла, VS Code интерпретирует этот файл как JavaScript и будет оценивать содержимое с помощью языковой службы JavaScript. Обратитесь к разделу «Язык кода JavaScript VS», чтобы узнать больше о поддержке JavaScript.

Создайте простую строковую переменную в app.jsи отправьте содержимое строки на консоль:

var msg = 'Hello World';
console.log(msg);

Обратите внимание , что , когда вы набрали console. IntelliSense на consoleобъекте был автоматически представлен Вам.

Также обратите внимание, что VS Code знает, что msgэто строка, основанная на инициализации 'Hello World'. Если вы напечатаете, msg.вы увидите IntelliSense, показывающий все строковые функции, доступные на msg.

После экспериментов с IntelliSense отмените любые дополнительные изменения из приведенного выше примера исходного кода и сохраните файл ( Ctrl + S ).

Запуск Hello World#

Это просто работать app.jsс Node.js. Из терминала просто наберите:

node app.js

Вы должны увидеть вывод «Hello World» на терминал, а затем Node.js вернется.

Интегрированный Терминал#

VS Code имеет встроенный терминал, который вы можете использовать для запуска команд оболочки. Вы можете запустить Node.js напрямую и избежать переключения кода VS во время работы инструментов командной строки.

Вид > Терминал ( Ctrl + ` с символом backtick) откроет интегрированный терминал, и вы можете запустить node app.jsтам:

Для этого пошагового руководства вы можете использовать внешний терминал или интегрированный терминал VS Code для запуска инструментов командной строки.

Отладка Hello World#

Как упоминалось во введении, VS Code поставляется с отладчиком для приложений Node.js. Давайте попробуем отладить наше простое приложение Hello World.

Чтобы установить точку останова app.js, поместите курсор редактора на первую строку и нажмите F9 или щелкните в левом желобе редактора рядом с номерами строк. Красный круг появится в желобе.

Чтобы начать отладку, выберите «Отладка» на панели действий:

Теперь вы можете нажать «Отладка» на зеленой панели инструментов или нажать F5, чтобы запустить и отладить «Hello World». Ваша точка останова будет достигнута, и вы сможете просмотреть и просмотреть простое приложение. Обратите внимание, что VS Code отображает другую цветную строку состояния, чтобы указать, что она находится в режиме отладки, и отображается консоль отладки.

Теперь, когда вы видели VS Code в действии с «Hello World», в следующем разделе показано использование VS Code с полнофункциональным веб-приложением Node.js.

Примечание. Мы закончили с примером «Hello World», поэтому перейдите из этой папки, прежде чем создавать приложение Express. Вы можете удалить папку «Hello», если хотите, так как она не требуется для остальной части пошагового руководства.

Экспресс приложение#

Express - это очень популярный фреймворк для создания и запуска приложений Node.js. Вы можете создать новое приложение Express с помощью инструмента Express Generator. Экспресс-генератор поставляется в виде модуля npm и устанавливается с помощью инструмента командной строки npm npm.

Совет: Чтобы проверить, что вы npmправильно установили на свой компьютер, введите npm --helpс терминала, и вы должны увидеть документацию по использованию.

Установите Express Generator, запустив в терминале следующее:

npm install -g express-generator

-gКоммутатор устанавливает генератор Экспресса во всем мире на вашем компьютере , так что вы можете запустить его из любой точки мира.

Теперь мы можем создать новое приложение Express, вызванное myExpressAppзапуском:

express myExpressApp --view pug

Это создаст новую папку myExpressAppс названием вашего приложения. В --view pugпараметрах говорят генератору использовать мопс двигатель шаблона (ранее известный как jade).

Чтобы установить все зависимости приложения (снова поставляются в виде модулей npm), перейдите в новую папку и выполните npm install:

cd myExpressApp
npm install

На данный момент мы должны проверить, что наше приложение работает. Сгенерированное приложение Express содержит package.jsonфайл, который содержит startскрипт для запуска node ./bin/www. Это запустит приложение Node.js.

Из терминала в папке приложения Express запустите:

npm start

Совет: Вы можете включить проводник для сценариев npm в своем рабочем пространстве, используя эту npm.enableScriptExplorerнастройку.

Запустится веб-сервер Node.js, и вы можете перейти по адресу http: // localhost: 3000, чтобы увидеть запущенное приложение.

Отличное редактирование кода#

Закройте браузер и из терминала в myExpressAppпапке, остановите сервер Node.js, нажав сочетание клавиш CTRL + C .

Теперь запустите VS Code:

code .

Примечание. Если вы использовали встроенный терминал VS Code для установки генератора Express и создания приложения, вы можете открыть myExpressAppпапку из запущенного экземпляра VS Code с помощью команды « Файл» > « Открыть папку» .

Документация Node.js и Express отлично справляется с объяснением того, как создавать многофункциональные приложения с использованием платформы и платформы. Visual Studio Code позволит вам более продуктивно разрабатывать приложения такого типа, предоставляя отличные возможности редактирования кода и навигации.

Откройте файл app.jsи наведите курсор мыши на глобальный объект Node.js __dirname. Обратите внимание, как VS Code понимает, что __dirnameэто строка. Еще интереснее то, что вы можете получить полную версию IntelliSense для платформы Node.js. Например, вы можете требовать httpи получать полную IntelliSense для httpкласса при вводе в коде Visual Studio.

VS Code использует файлы описания (типирования) TypeScript (например node.d.ts) для предоставления метаданных VS Code о каркасах на основе JavaScript, которые вы используете в своем приложении. Файлы объявления типов написаны на TypeScript, поэтому они могут выражать типы данных параметров и функций, что позволяет VS Code предоставлять богатый опыт IntelliSense. Благодаря функции Automatic Type Acquisition, которую вы называете , вам не нужно беспокоиться о загрузке этих файлов объявлений типов, VS Code установит их автоматически.

Вы также можете написать код, который ссылается на модули в других файлах. Например, app.jsнам требуется ./routes/indexмодуль, который экспортирует Express.Routerкласс. Если вы включите IntelliSense index, вы сможете увидеть форму Routerкласса.

Отладка вашего экспресс-приложения#

Вам нужно будет создать файл конфигурации отладчика launch.jsonдля вашего приложения Express. Щелкните значок «Отладка» на панели действий, а затем значок «Настроить механизм» в верхней части представления «Отладка», чтобы создать launch.jsonфайл по умолчанию . Выберите среду Node.js , указав для typeсвойства in configurationsзначение "node". Когда создается первый файл, VS код будет выглядеть package.jsonдля startсценария и будет использовать это значение в качестве program(который в данном случае "${workspaceFolder}\\bin\\www) для запуска программы конфигурации.

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "program": "${workspaceFolder}\\bin\\www"
    }
  ]
}

Сохраните новый файл и убедитесь, что Launch Program выбрана в раскрывающемся меню конфигурации в верхней части окна Debug. Откройте app.jsи установите точку останова в верхней части файла, в котором создается объект приложения Express, щелкнув мышью слева от номера строки. Нажмите F5, чтобы начать отладку приложения. VS Code запустит сервер в новом терминале и достигнет установленной нами точки останова. Оттуда вы можете проверять переменные, создавать часы и просматривать ваш код.
